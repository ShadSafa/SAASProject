Output: Cache service ready to store/retrieve analysis results, reducing API costs from estimated $20/day to $3/day.
</objective>

Create backend/app/services/cache_service.py implementing:

1. Redis client initialization:
```python
import redis
from datetime import timedelta
from app.config import settings

redis_client = redis.Redis.from_url(
    settings.CELERY_BROKER_URL,
    decode_responses=True  # Return strings not bytes
)
```

2. cache_analysis() function:
- Accept viral_post_id (int) and analysis (ViralAnalysisResult Pydantic model)
- Cache key: f"analysis:{viral_post_id}"
- Serialize: analysis.model_dump_json()
- Store with 7-day TTL: redis_client.setex(key, timedelta(days=7), value)
- Return None (fire-and-forget caching)

3. get_cached_analysis() function:
- Accept viral_post_id (int)
- Cache key: f"analysis:{viral_post_id}"
- Fetch: redis_client.get(key)
- If exists: deserialize with ViralAnalysisResult.model_validate_json(cached)
- If not exists: return None
- Handle Redis errors gracefully (return None on connection failure, log error)

4. clear_analysis_cache() function (for admin/testing):
- Accept viral_post_id (int)
- Delete key: redis_client.delete(f"analysis:{viral_post_id}")
