  <how-to-verify>
**Prerequisites:**
1. Ensure backend/.env has OPENAI_API_KEY set (get from https://platform.openai.com/api-keys)
2. Ensure Celery worker running: `cd backend && celery -A app.celery_app worker --loglevel=info --pool=solo`
3. Ensure Redis running (check via `redis-cli ping` -> should return "PONG")
4. Backend server running: `cd backend && uvicorn app.main:app --reload --port 8000`
5. Frontend dev server running: `cd frontend && npm run dev`

**Test Checklist:**

1. **ANALYSIS-01: "Why viral" summary generated**
   - [ ] Trigger a new scan (discover tab, 24h time range)
   - [ ] Wait for scan to complete (status: completed)
   - [ ] Click on any viral post card in results
   - [ ] Verify "Why It Went Viral" section appears with 2-3 sentence summary
   - [ ] Summary should be relevant to post content (not generic)

2. **ANALYSIS-02: Posting time analysis**
   - [ ] In same viral post, verify "Posting Time" badge shows score 0-100
   - [ ] Badge color should be green (>70) for prime time posts (6pm-10pm)
   - [ ] Badge color should be yellow/red for off-hours posts

3. **ANALYSIS-03/04: Hook strength (caption opening)**
   - [ ] Verify "Hook Strength" badge shows score 0-100
   - [ ] For posts with strong opening line (questions, emojis, intrigue), score should be >60
   - [ ] For weak openings, score should be <50

4. **ANALYSIS-05: Emotional trigger identification**
   - [ ] Verify "Emotional Trigger" badge shows emotion (joy/awe/anger/surprise/sadness/fear)
   - [ ] Emotion should match post content (e.g., funny content -> joy, shocking news -> surprise)

5. **ANALYSIS-07: Engagement velocity (already in Phase 3, but verify score)**
   - [ ] Verify "Engagement Velocity" badge shows score 0-100
   - [ ] High engagement posts (<24h old, 10k+ engagements) should score >80
   - [ ] Slower posts should score <50

6. **ANALYSIS-08: Save/share ratio**
   - [ ] Verify "Save/Share Ratio" badge shows score 0-100
   - [ ] Posts with high saves+shares (>20% of total engagement) should score >70
   - [ ] Posts with low saves+shares (<5%) should score <30

7. **ANALYSIS-09: Hashtag strategy analysis**
   - [ ] Verify "Hashtag Performance" badge shows score 0-100
   - [ ] Posts with 6-15 hashtags should score 60-90
   - [ ] Posts with 0 hashtags should score 0
   - [ ] Posts with 1-5 hashtags should score 20-60

8. **ANALYSIS-10: Audience retention (videos)**
   - [ ] Verify "Audience Retention" badge shows score 0-100
   - [ ] For video/reel posts, score should reflect content quality
   - [ ] For photo posts, N/A is acceptable

9. **Caching verification (cost optimization)**
   - [ ] Trigger second scan with same time range (should find duplicate posts)
   - [ ] Check backend logs: should see "Analysis cached" messages for duplicate posts
   - [ ] Verify analysis appears instantly for cached posts (no 30s wait)
   - [ ] Run `redis-cli keys "analysis:*"` — should show cached analysis keys

10. **Performance check**
    - [ ] New scan completes within 10 seconds (scan phase)
    - [ ] Analysis appears within 30 seconds after scan completes (background task)
    - [ ] UI remains responsive during analysis (no freezing)

11. **Error handling**
    - [ ] Disconnect internet briefly, trigger scan
    - [ ] Verify error message appears (not blank screen)
    - [ ] Reconnect, verify scan works again

12. **Confidence score**
    - [ ] In any analyzed post, verify confidence percentage shows (e.g., "Confidence: 85%")
    - [ ] Confidence should be 60-95% (realistic range)

**Success Criteria:**
- All 12 checklist items pass ✓
- Analysis results appear automatically after scan
- Cache reduces repeat analysis time from 30s to <1s
- UI displays all 7 algorithm factors with color coding
- OpenAI costs estimated at <$0.10 per scan (8 posts × $0.01-0.02 per analysis, minus cache hits)

**Known Limitations (acceptable for Phase 4):**
- Comment sentiment analysis implemented but not used (Phase 3 doesn't fetch comment text)
- Video first-frame analysis not implemented (requires OpenCV video processing, deferred if too complex)
- Hashtag performance is heuristic-based (count only, not trending analysis — OpenAI will refine)
