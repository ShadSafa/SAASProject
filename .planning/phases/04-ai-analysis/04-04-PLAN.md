  <action>
Create backend/app/services/algorithm_factors.py implementing:

1. calculate_engagement_velocity_score(viral_post: ViralPost) -> float:
- Formula: total_engagement / post_age_hours normalized to 0-100 scale
- total_engagement = likes_count + comments_count + saves_count + shares_count
- Normalize: min((total_engagement / post_age_hours) / 100, 1.0) * 100
- Handle zero division: if post_age_hours == 0, return 0.0
- Typical viral: 100 engagements/hour = score 100
- Round to 2 decimal places

2. calculate_save_share_ratio_score(viral_post: ViralPost) -> float:
- Formula: (saves_count + shares_count) / total_engagement * 100
- total_engagement = likes_count + comments_count + saves_count + shares_count
- Normalize: High save/share ratio (>20%) indicates high value content
- Score = min((save_share_count / total_engagement) * 500, 100.0)
- Handle zero division: if total_engagement == 0, return 0.0
- Round to 2 decimal places

3. calculate_hashtag_performance_score(hashtags: str | None) -> float:
- Parse hashtags (JSON array string or None)
- Count hashtags: 0 = score 0, 1-5 = linear scale 20-60, 6-15 = 60-90, 15+ = 90-100
- Simple heuristic (AI will refine in analyze_viral_post for trending/relevance)
- Return float 0-100

4. calculate_posting_time_score(created_at: datetime, creator_follower_count: int) -> float:
- Heuristic: Posts during 6pm-10pm local time (assuming UTC, use hour only) score higher
- Extract hour: created_at.hour
- If 18 <= hour <= 22: score 80-100 (prime time)
- If 12 <= hour < 18 or 22 < hour <= 24: score 50-80 (good time)
- If 6 <= hour < 12: score 30-50 (morning)
- If 0 <= hour < 6: score 10-30 (late night, lower engagement)
- Adjust +10 for large accounts (>100k followers) since they have global audiences
- Return float 0-100
