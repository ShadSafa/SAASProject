<objective>
Create Celery background tasks for AI analysis execution with cache integration.

Purpose: OpenAI API calls take 2-5 seconds per post. Running synchronously would block the FastAPI event loop and make the scan endpoint unresponsive. Background tasks enable fire-and-forget analysis while users continue using the app.

Output: Celery task that analyzes viral posts in batches, checks cache first, calls OpenAI on cache miss, stores results in DB and cache.
</objective>

2. analyze_posts_batch Celery task:
- Signature: @celery_app.task(name='analysis.analyze_posts_batch')
- Accept scan_id (int) and viral_post_ids (list[int])
- For each viral_post_id:
  a. Check cache: cached = get_cached_analysis(viral_post_id)
  b. If cached: create Analysis record from cached data, skip OpenAI call
  c. If not cached:
     - Fetch ViralPost from DB
     - Call analyze_viral_post(viral_post)
     - Create Analysis record from ViralAnalysisResult
     - Cache result: cache_analysis(viral_post_id, result)
  d. Handle errors per-post (try/except per post, log error, continue to next)
- Commit all Analysis records at end
- Return summary: {"analyzed": count, "cached": count, "failed": count}

3. Error handling:
- Wrap each post analysis in try/except
- Log OpenAI API errors but don't fail entire batch
- If all posts fail, log warning but task completes successfully
- Use asyncio.run() pattern from Phase 3 for async DB session (scan_jobs.py reference)

Use research pattern from 04-RESEARCH.md lines 159-211.
