## Problem Statement

**Gap 2**: NicheBadge component not properly wired to AnalysisPanel

The `NicheBadge` component in `AnalysisPanel.tsx` (lines 41-46) is called without the required `analysisId` and `userNicheOverride` props. This causes:
1. Runtime errors when NicheBadge tries to save niche overrides (missing `analysisId`)
2. Component cannot display user overrides (missing `userNicheOverride`)
3. User cannot customize niches (blocking ANALYSIS-19 requirement)

**Current broken code** (lines 41-46):
```tsx
<NicheBadge
  niche={analysis.niche || null}
  confidence={analysis.audience_interests?.niche_confidence || null}
  secondaryNiche={analysis.audience_interests?.niche_secondary || null}
  reasoning={analysis.audience_interests?.niche_reasoning || null}
/>
```

**Impact**: Niche override functionality completely broken despite backend API and NicheBadge component being fully functional.

**Root Cause**: When NicheBadge component was updated in plan 05-06 to support niche overrides, AnalysisPanel.tsx was not updated to pass the new required props.

## Solution

Update the NicheBadge component call in AnalysisPanel.tsx to pass the required `analysisId` and `userNicheOverride` props.
