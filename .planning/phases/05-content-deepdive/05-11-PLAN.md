## Problem Statement

**Gap 3**: Backend API response incomplete - missing Phase 05 enriched fields

The `get_analysis` endpoint in `backend/app/routes/analysis.py` (lines 55-68) returns a hardcoded dict that only includes Phase 04 OpenAI analysis fields. It does NOT return Phase 05 enriched fields:
- `engagement_rate`
- `content_category`
- `niche`
- `user_niche_override`
- `audience_demographics`
- `audience_interests`

**Current incomplete response** (lines 55-68):
```python
response = {
    "id": analysis.id,
    "viral_post_id": analysis.viral_post_id,
    "why_viral_summary": analysis.why_viral_summary,
    "posting_time_score": analysis.posting_time_score,
    "hook_strength_score": analysis.hook_strength_score,
    "emotional_trigger": analysis.emotional_trigger,
    "engagement_velocity_score": analysis.engagement_velocity_score,
    "save_share_ratio_score": analysis.save_share_ratio_score,
    "hashtag_performance_score": hashtag_score,
    "audience_retention_score": analysis.audience_retention_score,
    "confidence_score": analysis.confidence_score,
    "created_at": analysis.created_at.isoformat() if analysis.created_at else None,
}
```

**Impact**:
- Frontend receives `null` for all Phase 05 enrichment fields
- EngagementMetricsCard, ContentCategoryBadges, NicheBadge display nothing
- User niche override functionality broken (no data to display/edit)
- Phase 05 enrichment services run successfully but data never reaches frontend

**Root Cause**: The response dict was created in Phase 04 and never updated to include Phase 05 fields added in plans 05-01 through 05-06.

## Solution

Extend the response dict in `get_analysis` endpoint to include all Phase 05 enriched fields from the Analysis model.
